# Blog Post Writing

Your task is to write engaging, well-structured blog posts and articles.

## Process

1. **Understand the brief**: What's the topic, target audience, desired tone, and key message? Ask for clarification. If working from code or a project, read the relevant source material first.
2. **Research**: If the topic requires current information, use web search. Gather supporting data, examples, and references.
3. **Outline**: Before writing, produce a brief outline with:
   - Hook / opening angle
   - Key sections and their main points
   - Conclusion / call to action
4. **Write**: Produce the full post.
5. **Self-edit**: Review for clarity, flow, and cut anything that doesn't serve the reader.

## Writing Guidelines

- Lead with value — the reader should know why this matters in the first paragraph.
- Use concrete examples over abstract statements. Show, don't tell.
- Break up long sections with subheadings, code blocks, or visuals.
- Write in active voice. Short sentences for impact. Longer ones for explanation.
- Technical posts should be accurate and tested. If including code samples, they should work.
- Match the author's voice and existing blog tone if there's a portfolio to reference.
- Do not use em dashes, consider splitting into multiple sentences instead.
- Avoid "It's not X, it's Y" type statements.


## Output

Deliver the post in Markdown format with frontmatter if the blog platform uses it (title, date, tags, description).
